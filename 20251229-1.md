  四、关键技术点

  1. 钉钉基于支付宝：钉钉小程序使用 --type alipay 编译
  2. 环境检测：通过 typeof dd !== 'undefined' 判断钉钉环境
  3. 原生 API：钉钉环境下优先使用 my.request、my.arrayBufferToBase64 等原生方法
  4. Session 处理：验证码请求需保存 JSESSIONID，确保登录验证一致性

  四、问题与解决方案

| 问题              | 原因                            | 解决方案                                 |
| ----------------- | ------------------------------- | ---------------------------------------- |
| H5 页面显示 "~ /" | 缺少 index.html                 | 创建 src/index.html                      |
| H5 元素过大       | 样式未适配                      | 配置 pxtransform + rem 基准              |
| H5 验证码不显示   | Taro.arrayBufferToBase64 不支持 | 使用 window.btoa                         |
| H5 接口跨域       | 浏览器 CORS 限制                | 配置 devServer proxy                     |
| 钉钉验证码不显示  | arraybuffer 转换失败            | 使用 my.request + my.arrayBufferToBase64 |
| 钉钉验证码错误    | session 不一致                  | 用 my.request 获取并保存 session         |
