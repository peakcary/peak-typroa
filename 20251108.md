  🔄 业务逻辑架构

  1. 认证与授权流程 (axios.js → permission.js)

  用户登录
    ↓
  请求Token (SM2加密密码)
    ↓
  保存 access_token + refresh_token
    ↓
  自动刷新机制 (Axios拦截器)
    ├─ 401错误检测
    ├─ RefreshToken刷新
    └─ Token续期
    ↓
  权限控制 (路由beforeEach钩子)
    ├─ 检查Token有效性
    ├─ 锁屏功能
    └─ 菜单权限加载

  核心特性:
  - SM2非对称加密（密码加密）
  - AES对称加密（可选报文加密）
  - Token自动续期机制（防止过期中断）
  - 单一错误提示（避免重复弹窗）

  2. 权限体系 (store/modules/user.js)

  用户登录 → 获取权限信息 → 菜单树 + 按钮权限
       ↓
  Vuex State:
    - token/refreshToken
    - userInfo (用户信息)
    - roles (角色列表)
    - permission (权限码集合)
    - menu/menuAll (菜单树)
    ↓
  页面级权限检查 (routerMeta.isAuth)
  按钮级权限检查 (v-if="permission.xx_xx")

  3. CRUD通用逻辑 (mixins/crud.js)

  动态加载 API + Option配置
           ↓
  getList() ──→ 分页查询
  rowSave() ──→ 新增
  rowUpdate() → 编辑
  rowDel() ──→ 删除
  搜索/排序/分页事件处理
           ↓
  自动调用API并更新表格

  特点: 减少重复代码，统一数据流

  4. Avue框架集成

  Avue = Auto VUE（自动化Vue）
  - 配置化表单 <avue-form>
  - 配置化表单 + 表格 <avue-crud>
  - 配置化树 <avue-tree>
  - 自动生成表单字段、校验、格式化

  <!-- 示例：用户管理 -->
  <avue-crud
    :option="userOption"      <!-- 配置对象 -->
    :data="data"              <!-- 表格数据 -->
    v-model="form"            <!-- 表单数据绑定 -->
    @row-save="rowSave"       <!-- 保存事件 -->
    @row-update="rowUpdate"   <!-- 更新事件 -->
  />

  🚀 核心请求流程示例

  // 用户登录流程
  1. 密码使用SM2加密
  2. POST /blade-auth/login → 获取token
  3. 本地存储token (localStorage)
  4. 自动加载用户信息和菜单
  5. 路由权限检查
  6. 显示菜单和操作按钮

  // 数据请求流程
  1. API发送请求
  2. Axios拦截器添加:
     - Authorization header (Bearer token)
     - Blade-Requested-With (安全验证)
     - Accept-Language (国际化)
  3. 服务端返回数据
  4. 响应拦截器处理:
     - 401自动刷新token
     - 错误统一处理
     - 加密数据解密
  5. 返回给页面
