 修改原因详解

  一、H5 打包修改原因

  1. 安装热更新插件

  npm install @pmmmwh/react-refresh-webpack-plugin react-refresh -D
  原因：Taro 4.x 的 H5 开发模式默认启用 React 热更新，但依赖未自动安装，导致编译报错。

---
  2. 配置代理 (proxy)

  proxy: {
    '/api': { target: 'https://demo.hskj.cc', changeOrigin: true }
  }
  原因：浏览器有同源策略 (CORS)，H5 页面从 localhost:10086 请求 demo.hskj.cc 的接口会被浏览器拦截。代理让请求先到本地服务器，再由服务器转发，绕过浏览器限制。

---
  3. 创建 index.html

  原因：
  - 小程序有自己的入口机制，不需要 HTML
  - H5 是网页，必须有 HTML 文件作为入口
  - Taro 编译时需要这个模板来注入打包后的 JS/CSS

---
  4. 配置 rem 适配

  e.style.fontSize = (t / 750 * 20) + "px"
  原因：
  - 小程序用 rpx 单位自动适配屏幕
  - H5 用 px 是固定像素，不会自动缩放
  - 通过动态设置 font-size，让 rem 单位能按设计稿 750px 等比缩放

---
  5. H5 验证码转换用 window.btoa

  if (env === Taro.ENV_TYPE.WEB) {
    base64 = window.btoa(binary)
  }
  原因：Taro.arrayBufferToBase64 是小程序 API，浏览器中不存在。H5 环境需用浏览器原生的 window.btoa 方法。

---
  二、钉钉打包修改原因

  1. 添加类型声明 dingtalk.d.ts

  declare const dd: typeof dd
  declare const my: typeof my
  原因：
  - dd 和 my 是钉钉/支付宝小程序的全局对象
  - TypeScript 不认识这些变量，编译会报 "找不到名称 dd/my"
  - 声明文件告诉 TS 这些变量存在及其类型

---
  2. 创建 mini.project.json

  原因：
  - 这是支付宝/钉钉开发者工具识别项目的配置文件
  - 类似微信的 project.config.json
  - 没有这个文件，开发者工具无法正确打开项目

---
  3. 登录页面环境判断

  const isDingTalk = typeof dd !== 'undefined'
  原因：
  - 微信登录用 wx.login() + 手机号授权
  - 钉钉登录用 dd.getAuthCode()
  - 两者 API 完全不同，必须根据环境调用不同方法

---
  4. 验证码用 my.request 而非 Taro.request

  原因：

| 问题             | 说明                                                         |
| ---------------- | ------------------------------------------------------------ |
| arraybuffer 支持 | Taro.request 在钉钉环境下 responseType: 'arraybuffer' 可能不生效 |
| Session 一致性   | 验证码和登录必须用同一个 JSESSIONID，否则验证码校验失败      |
| Base64 转换      | Taro.arrayBufferToBase64 在钉钉中不可用，需用 my.arrayBufferToBase64 |

  使用原生 my.request 可以：
  1. 正确获取 arraybuffer 响应
  2. 从响应头提取并保存 session
  3. 用原生方法转换 base64

---
  三、核心原理总结

  ┌─────────────────────────────────────────────────────────┐
  │                    Taro 跨平台原理                        │
  ├─────────────────────────────────────────────────────────┤
  │  源代码 (React)                                          │
  │      ↓                                                   │
  │  Taro 编译器                                              │
  │      ↓                                                   │
  │  ┌──────────┬──────────┬──────────┬──────────┐          │
  │  │  微信     │   H5     │  支付宝   │   钉钉    │          │
  │  │  wx.*    │ 浏览器API │  my.*    │  dd.*    │          │
  │  └──────────┴──────────┴──────────┴──────────┘          │
  └─────────────────────────────────────────────────────────┘

  Taro 抹平了大部分差异，但某些场景需要手动适配：
  1. 平台特有 API（登录、支付）
  2. 底层实现差异（arraybuffer 处理）
  3. 运行环境差异（浏览器 vs 小程序容器）
