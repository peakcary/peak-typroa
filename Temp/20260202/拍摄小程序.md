景区旅拍小程序完善需求分析

  一、项目核心

  开发一款微信小程序，实现"游客扫码下单、后台派单、拍摄剪辑、线上交付素材"的全流程线上化。核心参考"飞享时光"等产品，优化体验。

  二、用户流程详解

  游客端流程：
  1. 扫码授权（头像授权及免责协议）
  2. 选景区/套餐
  3. 下单支付
  4. 看进度
  5. 收通知
  6. 下载素材
  7. 评价

  管理端流程：
  1. 接订单
  2. 派给工作人员
  3. 拍摄上传素材
  4. 剪辑后交付用户
  5. 处理售后

  工作人员端流程：
  1. 接单 → 确认（扫码）→ 拍摄 → 上传原始素材 → 填写交付信息

  三、核心功能模块

  1. 首页/景区模块

  用户端：
  - 微信一键登录
  - 自动定位推荐景区
  - 搜索并浏览景区列表与详情

  管理端：
  - 动态管理景区信息、套餐、价格
  - 配置上下线状态

  2. 服务与下单模块

  用户端：
  - 清晰展示四种服务（全景、航拍、跟拍、AR体验、定制）及套餐
  - 集成微信支付

  管理端：
  - 全面订单管理（查看、筛选、导出）
  - 智能派单给工作人员
  - 审核处理退款、免单

  3. 拍摄模式（新增）

  工作人员端：
  - 自动模式：
    - 无人机自动飞行预定义航线
    - 单次"起飞"命令触发自主操作
    - 自动拍摄并上传服务器
    - 实时状态更新（"飞行中"、"拍摄中"等）
    - 兼容"大树"平台（支持所有机型）
  - 自定义模式：
    - 工作人员手动拍摄
    - 编辑工具处理素材
    - 手动上传并推送

  4. 素材与订单模块

  用户端：
  - 实时查看订单状态
  - 接收微信通知
  - 预览并下载素材（素材服务器存储期为7天，到期前1天提醒，过期后自动删除，支持客服人工临时恢复）
  - 一键分享素材

  工作人员端：
  - 接单、扫码确认
  - 更新进度
  - 上传素材
  - 填写交付信息

  管理端：
  - 关联订单与素材，完成交付
  - 素材自动推送至小程序

  5. 评价与营销模块

  用户端：
  - 对已完成订单进行评分和图文评价
  - 浏览"热门作品"社区

  管理端：
  - 审核用户评价/作品
  - 发布优惠券
  - 数据统计（订单、用户、收入看板）

  四、技术要求

  性能要求：
  - 加载快速，运行流畅
  - 能支撑节假日访问高峰
  - 实时状态更新

  安全要求：
  - 用户数据加密
  - 支付安全
  - 素材安全存储

  集成要求：
  - 微信支付
  - 大树无人机平台集成
  - 微信消息推送

  交付要求：
  - 完整源码
  - 技术文档
  - 部署文档
  - 测试报告

  五、新增功能特性

  1. 智能派单系统

    - 自动分配订单给最近的工作人员
    - 考虑工作人员当前负载
    - 实时订单状态同步
  2. 无人机拍摄自动化

    - 预定义航线管理
    - 自动飞行和拍摄
    - 实时状态监控
    - 兼容多种无人机型号
  3. 素材自动处理

    - 服务器端自动优化图片
    - 自动推送至用户
    - 智能存储管理（7天存储期）
  4. 实时通知系统

    - 订单状态更新
    - 素材处理完成
    - 交付提醒
  5. 数据统计看板

    - 实时订单数据
    - 用户增长统计
    - 收入分析
    - 热门景区排行

  六、UI/UX要求

  1. 简洁直观

    - 一键扫码
    - 清晰的进度指示
    - 简单的操作流程
  2. 视觉反馈

    - 实时状态更新
    - 进度条显示
    - 成功/失败提示
  3. 响应式设计

    - 适配各种屏幕尺寸
    - 流畅的动画效果
    - 快速加载

  七、交付与支持

  - 交付物：完整源码、文档、测试报告
  - 上线支持：协助部署至上线
  - 免费售后：上线后提供一年免费维护（Bug修复、技术咨询）

  八、特殊说明

  1. 功能优先级

    - 一期：基础流程、手动拍摄模式、基本订单管理
    - 二期：无人机自动拍摄模式、高级数据分析
  2. 技术架构

    - 微信小程序多端（用户、管理、工作人员）
    - 后端API服务
    - 云存储服务
    - 消息推送服务
  3. 扩展性

    - 预留无人机接口
    - 支持未来新增服务类型
    - 可扩展的订单处理流程
