  🚀 建议的测试流程

  第一步：准备企业微信账号（15-20 分钟）
  1. 访问 https://work.weixin.qq.com 注册企业
  2. 获取企业 ID（在"我的企业"→"企业信息"）
  3. 创建自建应用，获取 AgentId 和 Secret
  4. 创建企业微信小程序，获取 AppID

  第二步：配置本地环境（2 分钟）
  编辑 server/.env 文件：
  WXWORK_CORP_ID="你的企业ID"
  WXWORK_CORP_SECRET="你的Secret"
  WXWORK_AGENT_ID="你的AgentId"
  REDIS_ENABLED="false"  # 测试环境不需要 Redis

  第三步：启动服务（2 分钟）
  # 启动后端
  cd server
  npm run start:dev

  # 启动管理后台
  cd admin
  npm run dev

  第四步：执行测试（20-30 分钟）
  1. 打开管理后台 http://localhost:9011
  2. 进入"企业微信同步"页面
  3. 点击"同步企业微信通讯录"
  4. 验证部门和成员同步成功
  5. 测试部门映射（自动或手动）
  6. 使用企业微信开发者工具测试小程序登录

  ⚠️ 重要提示

  1. 必须使用企业微信开发者工具（不是普通微信开发者工具）

    - 下载地址：https://developers.weixin.qq.com/miniprogram/dev/devtools/download.html
  2. Redis 是可选的

    - 测试环境设置 REDIS_ENABLED="false" 即可
    - 只有集群部署才需要 Redis
  3. 企业微信成员会自动通过审核

    - 状态会自动设置为 approved
    - 这是设计行为，因为企业微信已验证身份

  📞 遇到问题时

  查看文档的对应章节：
  - 配置问题 → 查看 WXWORK_TESTING_GUIDE.md 的"常见测试问题排查"
  - 同步失败 → 检查企业 ID、Secret、AgentId 配置
  - 登录失败 → 查看后端日志，搜索 [WXWORK] 关键字
  - 手机号无效 → 确保企业微信成员使用 11 位中国大陆手机号
